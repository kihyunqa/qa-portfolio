### BUG-001 · P1 · 로그아웃 후 토큰 여전히 유효 (보안)

| 항목 | 내용 |
|------|------|
| **버그 ID** | BUG-001 |
| **심각도** | P1 — 보안 |
| **상태** | ✅ 해결 완료 |
| **발견 방법** | 회귀 테스트 + Postman API 직접 호출 |
| **발견일** | 2026-03-30 |

**재현 방법:**
1. 로그인 후 Bearer 토큰 복사
2. UI에서 로그아웃 실행
3. 복사한 토큰으로 `/api/user/profile` 직접 호출
4. → **200 OK + 사용자 정보 반환** (기댓값: 401 Unauthorized)

**원인:** 로그아웃이 클라이언트 localStorage 삭제만 처리, 서버 토큰 무효화 미구현

**영향:** 공용 PC 환경, 탈취된 토큰으로 계정 무단 접근 가능

**해결:** Redis 블랙리스트 방식 서버 토큰 무효화 구현, 토큰 만료 5분 임시 단축

**QA 인사이트:** UI 로그아웃 통과 ≠ 보안 완료. 반드시 API 레벨까지 확인 필요.

### BUG-002 · P1 · 결제 소수점 버림으로 1원 오차 누적

| 항목 | 내용 |
|------|------|
| **버그 ID** | BUG-002 |
| **심각도** | P1 — 금전 오류 |
| **상태** | ✅ 해결 완료 |
| **발견 방법** | 경계값 테스트 (다중 상품 + 할인 조합) |
| **발견일** | 2026-03-30 |

**재현 방법:**
```
상품 A(13,333원) × 2 + 상품 B(9,999원) × 1, 10% 할인
기댓값: (26,666 + 9,999) × 0.9 = 32,999원
실제값: 33,000원 (1원 초과 청구)
```

**원인:** 상품별 할인 금액 계산 시 소수점 truncate 후 합산 → 오차 누적

**영향:** 대량 거래 시 건당 최대 수 원 오과금 누적 (전자금융거래법 이슈)

**해결:** 계산 순서 변경 (개별 계산 → 전체 합산 후 할인 1회 적용), 소수점 처리 정책 3자 합의 문서화

**QA 인사이트:** "1원이잖아요"에 흔들리지 않으려면 영향 규모(건수 × 금액)로 설명해야 한다.

### BUG-003 · P2 · 저속 네트워크 환경 알림 중복 발송

| 항목 | 내용 |
|------|------|
| **버그 ID** | BUG-003 |
| **심각도** | P2 — UX 품질 |
| **상태** | ✅ 해결 완료 |
| **발견 방법** | Chrome DevTools Network Throttling (Slow 3G) |
| **발견일** | 2026-03-30 |

**재현 방법:**
1. Chrome DevTools → Network → Slow 3G 설정
2. 주문 완료 액션 실행
3. → 동일 알림 2~3건 수신 (일반 네트워크에서는 미재현)

**원인:** 클라이언트 retry 로직 + 서버 멱등성 미처리 조합

**영향:** 모바일 저속 환경(지하철, 엘리베이터) 실 사용자 — 재현율 80%+

**해결:** 서버 idempotency key 적용, 클라이언트 exponential backoff 추가

**QA 인사이트:** 재현이 어렵다 = 테스트 조건이 좁다. 실 사용자 환경을 의도적으로 만들어야 한다.

### BUG-008 · P2 · 특수문자 포함 검색어 500 에러

| 항목 | 내용 |
|------|------|
| **버그 ID** | BUG-008 |
| **심각도** | P2 — 기능 오류 |
| **상태** | ✅ 해결 완료 |
| **발견 방법** | 입력값 경계값 테스트 |
| **발견일** | 2026-03-28 |

**재현 방법:**
1. 검색창에 `'; DROP TABLE users; --` 입력
2. → 500 Internal Server Error 반환
3. 또는 `%`, `_` 등 SQL 와일드카드 입력 시 예외 결과 반환

**원인:** 검색 쿼리 파라미터 입력값 sanitize 미처리, Prepared Statement 미사용

**영향:** SQL Injection 취약점 노출 — 보안 감사 시 즉시 지적 대상

**해결:** Prepared Statement 적용, 특수문자 이스케이프 처리, 입력값 길이 제한 추가

**QA 인사이트:** 검색 기능 TC에 특수문자/SQL 패턴은 기본 포함 항목. 보안 TC와 기능 TC를 분리하지 말 것.
