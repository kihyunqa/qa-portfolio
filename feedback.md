## 피드백 항목 & 처리 현황

### ✅ 1. 포트폴리오 이름 색상 흰색으로
- **피드백**: 히어로 섹션 이름(성기현)이 보라색인데 흰색으로 바꿔달라
- **처리 완료**: `h1.hero-name em { color:#ffffff }` 적용됨
- **파일**: `index.html` CSS 완료

### ✅ 2. 회색 글씨 가독성 향상
- **피드백**: 작은 회색 글씨들이 잘 안 보임. 글씨 키우고 색상 밝게
- **처리 완료**:
  - `--muted` 색상: `#d4cce8` (밝게 변경)
  - `--dim` 색상: `#b8aed4` (밝게 변경)
  - 각종 폰트 사이즈 업
- **파일**: `index.html` CSS 완료

### ✅ 3. 킥(한 방) 메시지 추가
- **피드백**: 내용은 많은데 뭘 잘하는지 모르겠다. 색깔이 보이는 강렬한 메시지가 필요
- **처리 완료**: 히어로 섹션에 킥 배너 추가
  - "QA가 느리다는 말 듣기 싫어서, AI로 직접 자동화했습니다."
  - 두 강점 카드 (6.9yr QA + 코드 0줄)
  - 기승전결 브릿지 6개 섹션 구분
- **추가 개선 필요**: 킥이 시각적으로 더 임팩트 있게 강화 필요
  - 히어로에 라이브 터미널 타이핑 추가 고려
  - 첫 눈에 "와!" 하는 시각적 충격이 부족

### 🔲 4. 이력서 vs 경력기술서 내용 차별화
- **피드백**: 이력서와 경력기술서 내용이 너무 비슷해서 같은 걸 2개 본 느낌
- **방향**:
  - **이력서** → 간결·압축형. 1페이지 분량 목표. 핵심 키워드 중심. 채용담당자가 30초에 파악할 수 있게
  - **경력기술서** → 상세·스토리형. 프로젝트별 구체적 성과 수치 포함. 어떤 문제를 어떻게 해결했는지 (문제→내가 한 것→결과 구조)
- **현재 상태**: 두 문서 모두 내용이 거의 동일한 구성
- **할 일**:
  1. 이력서: 각 경력사항을 3줄 이내로 압축. 스킬 섹션 앞으로 배치. 수치 강조
  2. 경력기술서: 각 프로젝트에 '문제 → 내가 한 것 → 결과' 구조로 재작성
  3. 경력기술서에 없는 내용(이유, 배경, 의사결정) 추가해 차별화

## 기술 메모

- 포트폴리오는 단일 `index.html` 파일 (CSS+JS 인라인)
- GitHub Pages로 자동 배포 (main 브랜치 push 시 5~10분 내 반영)
- CSS 변수: `--muted:#d4cce8`(본문 텍스트), `--dim:#b8aed4`(서브 텍스트), `--purple:#a855f7`(강조색), `--text:#f0eaff`(주요 텍스트)
- 이름 색상: `h1.hero-name em { color:#ffffff }` - 흰색 적용됨
- GitHub MCP로 파일 수정 후 push하면 5~10분 내 반영
- SHA 값이 필요하면 `github:get_file_contents`로 확인 후 사용
- index.html SHA 현재: `cc436fd53661a2899f425c134f91a6cc76781f05` (변경 시 업데이트 필요)
