## 피드백 항목 & 처리 현황

### ✅ 1. 포트폴리오 이름 색상 흰색으로
- **피드백**: 히어로 섹션 이름(성기현)이 보라색인데 흰색으로 바꿔달라
- **처리**: `h1.hero-name em { color: var(--text) }` → 흰색(#f0eaff)으로 변경
- **파일**: `index.html` CSS 수정 완료

### ✅ 2. 회색 글씨 가독성 향상
- **피드백**: 작은 회색 글씨들이 잘 안 보임. 글씨 키우고 색상 밝게
- **처리**:
  - `--muted` 색상: `#7c6f9a` → `#b8aed4` (훨씬 밝게)
  - `--dim` 색상: `#3d3358` → `#6b5f8a` (밝게)
  - `.sec-desc` font-size: 14px → 15px
  - `.tl-desc`, `.proj-desc`, `.proof-desc` font-size: 13px → 14px
  - `.mc-desc`, `.rm-desc` font-size: 11px → 12px
  - `.tc-intro` font-size: 14px → 15px
- **파일**: `index.html` CSS `:root` 변수 + 개별 클래스 수정 완료

### ✅ 3. 킥(한 방 메시지) 추가
- **피드백**: 내용은 많은데 뭘 잘하는지 모르겠다. 본인 색깔이 보이는 강렬한 메시지가 필요
- **처리**: 히어로 섹션 상단에 킥 배너 추가
  - "'QA가 느리다'는 말 듣기 싫어서, 자동화했습니다."
  - 서브 메시지로 강점 한 줄 요약
- **파일**: `index.html` hero 섹션 수정 완료

### 🔲 4. 이력서 vs 경력기술서 내용 차별화
- **피드백**: 이력서와 경력기술서 내용이 너무 비슷해서 같은 걸 2개 본 느낌
- **방향**:
  - **이력서** → 간결·압축형. 1페이지 분량 목표. 핵심 키워드 중심. 채용담당자가 30초에 파악할 수 있게
  - **경력기술서** → 상세·스토리형. 프로젝트별 구체적 성과 수치 포함. 어떤 문제를 어떻게 해결했는지
- **현재 상태**: 두 문서 모두 내용이 거의 동일한 구성
- **할 일**:
  1. 이력서: 각 경력사항을 3줄 이내로 압축. 스킬 섹션 앞으로 배치. 수치 강조
  2. 경력기술서: 각 프로젝트에 '문제 → 내가 한 것 → 결과' 구조로 재작성
  3. 경력기술서에 없는 내용(이유, 배경, 의사결정)을 추가해 차별화

## 기술 메모

- 포트폴리오는 단일 `index.html` 파일 (CSS+JS 인라인)
- GitHub Pages로 자동 배포 (main 브랜치 push 시 즉시 반영)
- CSS 변수: `--muted`(본문 텍스트), `--dim`(서브 텍스트), `--purple`(강조색)
- GitHub MCP로 파일 수정 후 push하면 5~10분 내 반영
- SHA 값이 필요하면 `github:get_file_contents`로 확인
