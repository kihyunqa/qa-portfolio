# 🚀 릴리즈 QA 프로세스

---

## 1. 릴리스 주기 (2주 스프린	기준)

```
1주쉠
  월: 요구사항 분석 + TC 설계
  화: 데븐 시작 후 TC 직접 검증
  수: 실행 + 버그 리포트
  목: 버그 피드백
  금: 시연 테스트 (버그 픽스 후)

2주쉠
  월: 통합 테스트 + 회귀 테스트
  화: E2E 실행 + 성능 측정
  수: 릴리스 후보 프리즈 승인
  목: 최종 철수 점검
  금: 프로덕션 배포 + 지켠 리포트
```

---

## 2. 입퇀/출퇀 기준

### 테스트 시작 조건 (Entry Criteria)
- [ ] 데모 환경 배포 완료
- [ ] TC 승인 완료
- [ ] 테스트 데이터 준비 완료
- [ ] 블로커 버그 없음

### 릴리스 승인 조건 (Exit Criteria)
- [ ] P1 버그 0건
- [ ] P2 버그 5건 이하
- [ ] TC PASS율 80% 이상
- [ ] 성능 기준치 충족 (3초 이하)
- [ ] 보안 TC 진가 승인
- [ ] 접근성 위반 없음

## 3. QA 콌스에서 트리거되는 자동화

| 이벤트 | 자동 동작 |
|--------|----------|
| TC 파일 push | GitHub Actions → Slack 알림 |
| PR merge | 테스트 실행 + 리포트 |
| 릴리스 | Notion TC DB 업데이트 |
| 형상 버그 | Slack P1 긴급 알림 |
