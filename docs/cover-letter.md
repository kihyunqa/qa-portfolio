## 버전 A — MCP 자동화 강조형

안녕하세요. QA 엔지니어 성기현입니다.

저는 6년 9개월간 삼성전자 파트너사, 모빌리티 스타트업, QA 파트 리더를 거치며 **테스트케이스 설계부터 E2E 자동화, 팀 리딩까지** 전방위 QA 역량을 쌓아왔습니다.

최근에는 여기에 **Claude MCP 자동화**를 더했습니다. filesystem · playwright · github · notion · slack, 5개 MCP 서버를 실제로 연결하여 **대화 한 번으로 TC 생성 → E2E 실행 → GitHub 커밋 → Notion 문서화 → Slack 알림까지 전부 자동화**하는 파이프라인을 직접 구축했습니다. 코드는 한 줄도 쓰지 않았습니다.

이 경험은 단순히 "AI 툴을 써봤다"는 수준이 아닙니다. **QA 워크플로우 전체를 자동화 파이프라인으로 재설계**한 경험이며, 팀에 즉시 적용 가능한 실용적인 스킬입니다.

함께 일하게 된다면, 반복적인 QA 업무를 자동화하고 팀이 더 중요한 판단에 집중할 수 있도록 기여하겠습니다.

## 버전 B — 경력 강조형

안녕하세요. 6년 9개월 경력의 QA 엔지니어 성기현입니다.

**삼성전자 파트너사(모비프렌)**에서는 IoT SmartThings, AI Bixby, 삼성 모바일 앱의 디바이스 QA를 담당하며 다양한 환경에서의 품질 검증 역량을 키웠습니다.

**IMS Mobility**에서는 모바일앱·웹·백오피스 QA를 통합 수행하며, Cypress E2E 자동화 스크립트를 직접 작성하고 Swagger 기반 API 테스트를 운영했습니다. 결제 QA를 전담하며 서비스 오픈 전 치명적 버그 다수를 사전 차단한 경험이 있습니다.

**두플**에서는 QA Part Leader로서 테스트 전략 수립, TC 설계, 팀 리딩을 담당했습니다. 이 시기에 Claude MCP를 도입하여 반복 업무를 자동화하고, QA 파이프라인 전체를 재구조화했습니다.

저는 **품질은 테스트가 아니라 설계에서 나온다**고 믿습니다. 좋은 TC 하나가 수십 번의 수동 테스트를 대체하고, 좋은 자동화 파이프라인 하나가 팀 전체의 생산성을 바꿉니다. 그런 변화를 함께 만들고 싶습니다.
