## 1️⃣ LinkedIn — 포스트 올리기

### 추천 순서

1. **docs/linkedin-post.md** 열기
2. 상황에 맞는 버전 선택:

| 내 상황 | 선택할 버전 |
|---------|-----------|
| 처음 포트폴리오 공개 | **버전 1** (임팩트형 ⚡) |
| QA 동료 네트워크에 공유 | **버전 2** (스토리형 📖) |
| 채용담당자 눈에 띄고 싶다 | **버전 3** (채용담당자용 📋) |
| 개발자/DevOps 커뮤니티 | **버전 4** (GitHub Actions 강조 ⚙️) |
| 이직 중임을 공개적으로 표명 | **버전 5** (구직 활동 중 🎯) |

3. 텍스트 복사 → LinkedIn 게시물 작성창에 붙여넣기
4. 이미지 첨부 권장 (스크린샷이 있으면 클릭률 약 2~3배 향상)
5. 게시

### 최적 게시 타이밍
- **요일**: 화~목 (월요일은 피드 경쟁 심함)
- **시간**: 오전 8~10시 or 오후 12~2시 (KST)
- **첫 댓글**: 게시 직후 본인이 댓글로 "포트폴리오 링크: ..." 추가하면 알림 도달률 상승

## 3️⃣ 직접 지원 — 커버레터 발송

### 단계별 실행

**Step 1**: `docs/cover-letter.md` 열기  
**Step 2**: 버전 선택 매트릭스 참고해서 버전 결정  
**Step 3**: `{Company}`, `{JD 키워드}`, `{포지션명}` 교체  
**Step 4**: 이메일 제목 형식:

```
[지원] QA Engineer 성기현 | 6년 9개월 경력 | MCP 자동화 포트폴리오
```

또는

```
[지원] {포지션명} 성기현 — 포트폴리오 첨부
```

**Step 5**: 이메일 본문에 커버레터 + 포트폴리오 링크 포함하여 발송
