## Bug Report 작성 양식

```
[BUG-XXXX] 버그 제목을 간결하게 작성

## 기본 정보
- 발견일: YYYY-MM-DD
- 발견자: (이름)
- 심각도: Critical / Major / Minor / Trivial
- 우선순위: P0 / P1 / P2 / P3
- 상태: Open / In Progress / Fixed / Closed
- 버전: v1.0.0
- 환경: Chrome 120 / Windows 11 / 1920x1080

## 재현 단계 (Steps to Reproduce)
1. URL 접속: https://...
2. (동작 1)
3. (동작 2)
4. 결과 확인

## 기대 결과 (Expected Result)
> 정상적으로 동작했을 때 나와야 할 결과

## 실제 결과 (Actual Result)
> 현재 실제로 나오는 결과 (버그 내용)

## 스크린샷 / 로그
[스크린샷 첨부 또는 콘솔 에러 로그]

## 재현율
- [ ] 100% 재현
- [ ] 간헐적 (약 __%)
- [ ] 1회 발생

## 추가 메모
(관련 이슈, PR, 원인 추정 등)
```

## 실제 버그 리포트 예시

### [BUG-001] 로그인 후 세션 유지 실패 (새로고침 시 로그아웃)

| 항목 | 내용 |
|------|------|
| 발견일 | 2026-03-01 |
| 심각도 | Major (P1) |
| 상태 | Fixed |
| 환경 | Chrome 120, Windows 11 |

**재현 단계:**
1. https://example.com/login 접속
2. 유효한 아이디/비밀번호로 로그인
3. 로그인 성공 후 메인 페이지 진입 확인
4. F5 (새로고침) 실행

**기대 결과:** 새로고침 후에도 로그인 상태 유지

**실제 결과:** 새로고침 시 로그인 페이지로 리다이렉트, 세션 초기화

**원인:** SessionStorage 대신 Cookie 미사용 → 탭 새로고침 시 휘발

### [BUG-002] 검색창 특수문자 입력 시 500 에러

| 항목 | 내용 |
|------|------|
| 발견일 | 2026-03-05 |
| 심각도 | Critical (P0) |
| 상태 | Fixed |
| 환경 | 전 브라우저 공통 |

**재현 단계:**
1. 메인 페이지 검색창 클릭
2. `<script>alert(1)</script>` 입력
3. 엔터 또는 검색 버튼 클릭

**기대 결과:** 입력값 이스케이프 처리 후 정상 검색 또는 에러 메시지 표시

**실제 결과:** Internal Server Error 500 반환, 서버 로그에 SQL 오류 노출

**원인:** 입력값 sanitization 미적용, SQL injection 취약점 동반 발견

### [BUG-003] 모바일 뷰 장바구니 버튼 클릭 불가

| 항목 | 내용 |
|------|------|
| 발견일 | 2026-03-10 |
| 심각도 | Minor (P2) |
| 상태 | Open |
| 환경 | iOS Safari 17, iPhone 15 |

**재현 단계:**
1. iPhone Safari에서 상품 상세 페이지 접속
2. 하단 '장바구니 담기' 버튼 탭

**기대 결과:** 장바구니에 상품 추가

**실제 결과:** 버튼 반응 없음 (클릭 이벤트 미발생)

**원인 추정:** z-index 레이어 충돌로 투명 오버레이가 버튼을 덮고 있음
