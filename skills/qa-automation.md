# QA 자동화 Skill 지침

> Claude Code Skill 파일 — QA 엔지니어링 전용  
> 이 파일을 Claude Code에 로드하면 QA 자동화 작업에 일관된 방식이 적용됩니다.

---

## 📋 기본 원칙

1. **TC 생성 시**: 해피패스 → 네거티브 → 엣지케이스 순으로 분류
2. **파일 저장 시**: `C:/qa-portfolio/` 하위에 `testcase_{기능명}.md` 형식
3. **GitHub 커밋 시**: `feat:`, `fix:`, `docs:` 컨벤션 준수
4. **버그 리포팅 시**: 심각도(P1~P4) + 재현 단계 + 기대/실제 결과 포함

---

## 🤖 TC 생성 템플릿

```markdown
## TC-{구분}-{번호} · {제목}

**구분**: 해피패스 / 네거티브 / 엣지케이스  
**우선순위**: P1 / P2 / P3  
**전제조건**: ...  

### 테스트 단계
1. ...
2. ...

### 기대 결과
- ...

### 실제 결과
- [ ] 미실행
```

---

## 🐛 버그 리포트 템플릿

```markdown
## BUG-{번호} · {제목}

**심각도**: P1 CRITICAL / P2 MAJOR / P3 MINOR / P4 TRIVIAL  
**발견 환경**: ...

### 재현 단계
1. ...

### 기대 결과 vs 실제 결과
- 기대: ...
- 실제: ...

### 스크린샷
(playwright MCP로 자동 첨부)
```

## ⚡ MCP 활용 규칙

| 작업 | 사용 MCP |
|---|---|
| TC 파일 저장 | filesystem MCP |
| 브라우저 테스트 | playwright MCP |
| 코드 커밋 | github MCP |
| 버그 조회/등록 | jira MCP |
| 리포트 작성 | confluence MCP |
| 팀 알림 | slack MCP |
