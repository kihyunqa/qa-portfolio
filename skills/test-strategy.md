# 🎯 Test Strategy & QA Process

> QA Engineer - kihyunqa | QA 전략 및 프로세스 문서

---

## 1. QA 플로우 전체 맵

```
요구사항 분석
    ↓
테스트 케이스 설계 (TC)
    ↓
수동 테스트 실행
    ↓
자동화 코드 작성 (Playwright)
    ↓
CI/CD 파이프라인 연동
    ↓
결합 결합 컴파일리어스 + Slack 알림
    ↓
레트로스펙티브 + 개선
```

## 2. 테스트 피라미드

```
          /\
         /  \
        / E2E\ (10%)
       /------\
      / Integr \ (20%)
     /----------\
    /   Unit     \ (70%)
   /────────────\
```

| 레이어 | 비율 | 도구 | 대상 |
|--------|------|------|------|
| Unit | 70% | Jest | 함수, 컴포넌트 |
| Integration | 20% | Playwright API | API 연동 |
| E2E | 10% | Playwright | 전체 유저 플로우 |

## 3. 리스크 기반 테스트 우선순위

| 위험도 | 빡도 | 우선순위 | 예시 |
|--------|------|----------|------|
| 높음 | 높음 | P1 (즐시) | 로그인, 결제 |
| 높음 | 낙음 | P2 (당일) | 검색, 장바구니 |
| 낙음 | 높음 | P3 (주중) | 프로필, 설정 |
| 낙음 | 낙음 | P4 (수시) | UI 세부 교정 |

## 4. 쫐 주간 QA 스프린	

| 요일 | 활동 |
|------|------|
| 월 | TC 설계 + 코드 리뷰 |
| 화 | 수동 TC 실행 |
| 수 | 자동화 TC 실행 + 결합 업데이트 |
| 목 | 버그 리포트 + 기록 |
| 금 | 레트로스펙티브 + Slack 리포트 |

## 5. KPI 지표

| 지표 | 목표값 | 현재 |
|------|--------|------|
| TC 컴버리지 | 80%이상 | 100% |
| 버그 발견 안 | 100% 처리 | 100% |
| 자동화 비율 | 70%이상 | Playwright 56건 |
| CI/CD 연동 | 완료 | GitHub Actions |
| 컴플리언스 알림 | 실시간 | Slack |
