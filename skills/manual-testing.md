## 2. 테스트 케이스 설계 기법

### Equivalence Partitioning (동치 분할)
- 유효 값 범위와 무효 값 범위 구분
- 예: 로그인 비밀번호 8~16자 → 가이더≤7, 8~16, 17개이상 3가지 구분

### Boundary Value Analysis (경계값 분석)
- 경계 값 -1, 경계값, 경계값+1 테스트
- 예: 한도 100일 륜 시, 99/100/101일 테스트

### Decision Table (의사결정 테이블)
- 조건 조합별 예상 결과 정리
- 브랜치 커버리지 확보

### State Transition (상태 전환)
- 상태다이어그램 기반 플로우 테스트
- 예: 주문 상태 (draft → pending → paid → shipped → delivered)

## 3. 버그 리포트 작성 기준

```markdown
### BUG-[ID]
- 제목: 짧고 명확하게
- 심각도: Critical / High / Medium / Low
- 재현 단계: 연번으로 작성
- 예상 결과: 감지뾯이 기대하는 것
- 실제 결과: 실제 일어난 것
- 스크린샷/로그: 첨부 증거
- 환경: OS/브라우저/버전
```
