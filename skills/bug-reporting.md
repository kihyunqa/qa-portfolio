# 🐛 Bug Reporting Skills

> 효과적인 버그 리포트 작성 역량

---

## 버그 리포트 핵심 원칙

1. **재현 가능성 확보** — 100% 재현 가능한 단계 명시
2. **환경 정보 상세 기록** — OS, 브라우저, 버전, 해상도
3. **스크린샷/영상 첨부** — 시각적 증거 필수
4. **심각도와 우선순위 구분** — P0~P3 기준 일관 적용
5. **기대 결과 vs 실제 결과** — 명확한 대비 기술

---

## 심각도 분류 기준

| Level | 조건 | 대응 시간 | 예시 |
|-------|------|-----------|------|
| P0 Critical | 서비스 불가 | 즉시 (4h) | 결제 완전 불가 |
| P1 Major | 핵심 기능 장애 | 24h | 로그인 불가 |
| P2 Minor | 부분 기능 저하 | 1주 | 검색 필터 오작동 |
| P3 Trivial | 미관상 이슈 | 다음 릴리즈 | 여백 1px 불일치 |

---

## 실제 발견한 주요 버그 유형

- 세션 만료 처리 미흡 (P1) — 새로고침 시 로그아웃
- 검색 특수문자 500 에러 (P0) — XSS + SQL Injection 동반
- iOS Safari 결제 버튼 미반응 (P2) — z-index 충돌
- 모바일 장바구니 수량 0 입력 허용 (P2) — 유효성 검사 누락
