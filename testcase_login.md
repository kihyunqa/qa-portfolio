# 로그인 기능 테스트케이스

## 기능 개요

| 항목 | 내용 |
|------|------|
| 기능명 | 로그인 |
| 인증 방식 | 이메일 + 비밀번호 |
| 성공 동작 | 메인 화면으로 이동 |
| 실패 동작 | 에러 메시지 표시 |
| 잠금 조건 | 5회 연속 실패 시 계정 잠금 |

---

## ✅ 해피 패스 (Happy Path)

| TC-ID | 테스트 제목 | 사전 조건 | 입력값 | 기대 결과 | 우선순위 |
|-------|-----------|---------|-------|---------|---------|
| TC-HP-001 | 정상 이메일/비밀번호로 로그인 성공 | 가입된 계정 존재 | 이메일: user@example.com / PW: Valid1234! | 메인 화면으로 이동, 로그인 세션 생성 | P1 |
| TC-HP-002 | 이메일 대소문자 무관 로그인 성공 | 가입된 계정 존재 | 이메일: USER@EXAMPLE.COM / PW: Valid1234! | 메인 화면으로 이동 | P2 |
| TC-HP-003 | 로그인 성공 후 이전 페이지로 리다이렉트 | 비로그인 상태에서 보호 페이지 접근 시도 | 정상 자격증명 입력 | 로그인 전 접근하려던 페이지로 이동 | P2 |
| TC-HP-004 | 계정 잠금 해제 후 정상 로그인 | 잠금 해제된 계정 | 이메일: locked@example.com / PW: Valid1234! | 메인 화면으로 이동 | P2 |

## ❌ 네거티브 케이스 (Negative Case)

| TC-ID | 테스트 제목 | 사전 조건 | 입력값 | 기대 결과 | 우선순위 |

| TC-NG-001 | 존재하지 않는 이메일로 로그인 시도 | - | 이메일: notexist@example.com / PW: Any1234! | "이메일 또는 비밀번호가 올바르지 않습니다" 에러 메시지 표시 | P1 |
| TC-NG-002 | 잘못된 비밀번호로 로그인 시도 | 가입된 계정 존재 | 이메일: user@example.com / PW: WrongPass! | "이메일 또는 비밀번호가 올바르지 않습니다" 에러 메시지 표시 | P1 |
| TC-NG-003 | 이메일 미입력 후 로그인 시도 | - | 이메일: (공백) / PW: Valid1234! | "이메일을 입력해 주세요" 유효성 에러 표시, 로그인 요청 미발송 | P1 |
| TC-NG-004 | 비밀번호 미입력 후 로그인 시도 | - | 이메일: user@example.com / PW: (공백) | "비밀번호를 입력해 주세요" 유효성 에러 표시, 로그인 요청 미발송 | P1 |
| TC-NG-005 | 이메일/비밀번호 모두 미입력 | - | 이메일: (공백) / PW: (공백) | 두 필드 모두 유효성 에러 표시 | P1 |
| TC-NG-006 | 이메일 형식 오류 | - | 이메일: userexample.com / PW: Valid1234! | "올바른 이메일 형식을 입력해 주세요" 에러 표시 | P1 |
| TC-NG-007 | 5회 연속 로그인 실패 → 계정 잠금 | 가입된 계정 존재, 실패 이력 없음 | 잘못된 비밀번호로 5회 반복 입력 | 5회째 실패 후 계정 잠금, "계정이 잠겼습니다" 메시지 표시 | P1 |
| TC-NG-008 | 계정 잠금 상태에서 로그인 시도 | 계정 잠금 상태 | 이메일: locked@example.com / PW: Valid1234! | "계정이 잠겼습니다. 관리자에게 문의하세요" 에러 메시지 표시 | P1 |
| TC-NG-009 | 탈퇴한 계정으로 로그인 시도 | 탈퇴 처리된 계정 존재 | 탈퇴 계정 자격증명 입력 | "이메일 또는 비밀번호가 올바르지 않습니다" 에러 표시 (탈퇴 여부 노출 금지) | P2 |
| TC-NG-010 | 비밀번호에 XSS 스크립트 입력 | - | PW: `<script>alert(1)</script>` | 스크립트 실행 없이 로그인 실패 에러 처리 | P1 |

## ⚠️ 엣지 케이스 (Edge Case)

| TC-ID | 테스트 제목 | 사전 조건 | 입력값 | 기대 결과 | 우선순위 |
|-------|-----------|---------|-------|---------|---------|
| TC-EC-001 | 이메일 앞뒤 공백 포함 입력 | 가입된 계정 존재 | 이메일: " user@example.com " (공백 포함) | 공백 trim 처리 후 정상 로그인 또는 에러 처리 (정책 확인 필요) | P2 |
| TC-EC-002 | 비밀번호 최대 길이 경계값 입력 | - | PW: 128자 이상 문자열 | 최대 길이 초과 안내 또는 정상 처리 (정책에 따름) | P2 |
| TC-EC-003 | 4회 실패 후 1회 성공 → 실패 카운트 초기화 | 가입된 계정, 실패 4회 상태 | 5회째에 올바른 비밀번호 입력 | 로그인 성공 및 실패 카운트 초기화, 계정 잠금 미발생 | P1 |
| TC-EC-004 | 4회 실패 후 1회 성공 → 재실패 시 카운트 기준 | 가입된 계정, 실패 카운트 초기화 상태 | 이후 다시 5회 연속 실패 | 5회 실패 후 계정 잠금 정상 동작 | P2 |
| TC-EC-005 | 동일 계정 동시 다중 로그인 시도 | 가입된 계정 존재 | 두 브라우저에서 동시에 로그인 요청 | 정책에 따라 중복 세션 허용 또는 기존 세션 만료 처리 | P2 |
| TC-EC-006 | 뒤로가기로 로그인 페이지 재진입 | 로그인 성공 후 메인 화면 | 브라우저 뒤로가기 버튼 클릭 | 이미 로그인 상태이므로 메인 화면 유지 또는 로그인 페이지 캐시 미노출 | P2 |
| TC-EC-007 | 네트워크 단절 상태에서 로그인 시도 | 네트워크 오프라인 | 정상 자격증명 입력 후 제출 | "네트워크 연결을 확인해 주세요" 또는 적절한 에러 메시지 표시 | P2 |
| TC-EC-008 | SQL Injection 패턴 입력 | - | 이메일: `' OR '1'='1` / PW: 임의 | 쿼리 조작 없이 로그인 실패 처리, 서버 에러 미노출 | P1 |
| TC-EC-009 | 이메일 최대 길이 경계값 입력 | - | 이메일: 255자 이상 문자열@example.com | 최대 길이 초과 안내 또는 정상 처리 | P3 |
| TC-EC-010 | 자동완성 비밀번호 사용 후 로그인 | 브라우저 비밀번호 저장됨 | 브라우저 자동완성으로 비밀번호 입력 | 자동완성 값으로 정상 로그인 처리 | P3 |
