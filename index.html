STORY-001 · 보안 P1 · 출시 차단
🔐 로그아웃 후 토큰이 여전히 유효
회귀 테스트 중 로그아웃 후 복사해둔 Bearer 토큰으로 API를 재호출했더니 200 OK가 반환됐습니다. UI 로그아웃은 localStorage 삭제뿐이었고, 서버 측 토큰 무효화가 빠져 있었습니다.
1. Postman으로 로그인 → Bearer 토큰 복사
2. UI에서 로그아웃 실행
3. 복사한 토큰으로 /api/user/profile 직접 호출
4. → 200 OK + 사용자 정보 반환 ← 이상 발견
QA 인사이트: 로그아웃은 UI 동작만 확인하면 통과처럼 보이지만, 실제 보안은 서버 측에서 토큰을 죽이는지가 핵심입니다. API 레벨 검증 없이는 절대 잡을 수 없는 버그.
PostmanAPI 검증보안 테스트Redis 블랙리스트재현율 100%
STORY-002 · 결제 P1 · 에스컬레이션
💳 결제 소수점 1원 오차 누적
3개 이상 상품 혼합 구매 + 할인율 적용 시 최종 결제 금액이 장바구니 합산보다 1원 초과 청구됐습니다. 개발팀이 단순 반올림 이슈로 처리하려 했으나 계산 순서 자체의 설계 문제임을 설명하고 P1으로 에스컬레이션했습니다.
상품 A(13,333원)×2 + 상품 B(9,999원)×1, 10% 할인
기대: (13,333×2 + 9,999) × 0.9 = 32,999원
실제: 33,000원 (1원 초과 청구)
원인: 상품별 할인 계산 → 소수점 truncate → 합산
QA 인사이트: 1원 차이는 작아 보이지만 대량 거래 시 수천 건 × 1원 = 수천 원 오과금. "의심스러운 케이스를 끝까지 파고드는 습관"이 이 버그를 잡았습니다.
경계값 테스트조합 테스트결제 QA에스컬레이션TC 12건 추가
STORY-003 · UX P2 · 조건부 허용
🔔 저속 네트워크에서 알림 2~3건 중복
