# E2E 시나리오: 결제 플로우 자동화

> 생성일: 2026-03-30 | 도구: Claude + playwright MCP

---

## 🎯 테스트 목적

장바구니 → 결제 → 완료까지 전체 결제 플로우 E2E 검증

---

## 🧪 시나리오 1: 정상 결제 플로우

| 단계 | 액션 | 기대결과 |
|------|------|----------|
| 1 | 상품 상세 페이지 접속 | 페이지 로드 완료 |
| 2 | "장바구니 담기" 클릭 | 장바구니 카운트 +1 |
| 3 | 장바구니 페이지 이동 | 상품 목록 표시 |
| 4 | "주문하기" 클릭 | 결제 페이지 이동 |
| 5 | 배송지 입력 | 주소 필드 채워짐 |
| 6 | 카드 정보 입력 | 카드 필드 채워짐 |
| 7 | "결제 완료" 클릭 | 주문 완료 페이지 이동 |
| 8 | 주문번호 확인 | 주문번호 텍스트 표시 |

**결과:** ✅ PASS

---

## 🧪 시나리오 2: 재고 없는 상품 결제 시도

| 단계 | 액션 | 기대결과 |
|------|------|----------|
| 1 | 품절 상품 페이지 접속 | 페이지 로드 |
| 2 | "장바구니 담기" 버튼 상태 확인 | 버튼 비활성화 표시 |
| 3 | 버튼 클릭 시도 | 클릭 불가 또는 에러 메시지 |
| 4 | "품절" 텍스트 확인 | 품절 안내 표시 |

**결과:** ✅ PASS

---

## 🧪 시나리오 3: 결제 실패 처리

| 단계 | 액션 | 기대결과 |
|------|------|----------|
| 1 | 결제 페이지 접속 | 페이지 로드 |
| 2 | 유효하지 않은 카드번호 입력 | 입력 허용 |
| 3 | "결제 완료" 클릭 | 결제 실패 메시지 표시 |
| 4 | 에러 메시지 확인 | "카드 정보를 확인하세요" 표시 |
| 5 | 재입력 가능 여부 확인 | 필드 초기화 또는 수정 가능 |

**결과:** ✅ PASS
